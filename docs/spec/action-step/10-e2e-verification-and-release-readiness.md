# Step 10：整合驗收與發佈就緒

## 目標

完成端到端驗證、品質門檻與打包更新流程，讓系統可穩定交付。

## 前置依賴

- `01` 至 `09` 全部完成

## 實作清單

1. 建立 E2E 測試情境（至少）：
   - 正常路徑：截圖→OCR→解析→疊加→預覽→儲存
   - Skip Preview 路徑
   - OCR 失敗與解析不完整路徑
   - 主視窗關閉後常駐系統匣路徑
   - Tray 右鍵 `關閉` 真正退出路徑
   - Tray 雙擊與 `設定` 開啟 Config 視窗路徑
2. 準備測試資料集：
   - 高品質影像
   - 低對比影像
   - 噪點影像
3. 建立驗收指標：
   - 解析正確率
   - 單次流程耗時
   - 失敗可恢復率
4. 打包與更新流程：
   - `electron-builder` 打包
   - `electron-updater` 對接 GitHub Releases
5. 建立發佈清單（Release Checklist）：
   - 版本號、變更摘要、回滾方案、已知限制

## 交付物

- E2E 測試報告
- 打包產物與更新設定
- Release Checklist 文件

## 驗收標準

- 三條關鍵流程（正常/Skip/OCR異常）均通過。
- 產出安裝包可在目標環境啟動並執行完整流程。
- 新版可透過自動更新機制被偵測與更新。
- 關閉主視窗時程式不退出，且可由系統匣重新喚回 Config 視窗。
- 僅 Tray 選單 `關閉` 可終止程式，行為與規格一致。
